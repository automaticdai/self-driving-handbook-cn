# 百度 Apollo

百度 Apollo 是中国最具影响力的自动驾驶开放平台，于 2017 年 4 月正式发布，以"阿波罗登月计划"为名，寓意用开放合作推动中国自动驾驶技术腾飞。Apollo 采用"开放能力、共建生态"的策略，向行业开放感知、规划、控制等核心模块，同时积累商业化运营数据。


## 平台演进历程

| 版本 | 发布时间 | 核心里程碑 |
| --- | --- | --- |
| Apollo 1.0 | 2017.07 | 简单循环路线封闭测试，RTK GPS 定位，开放控制模块（LQR） |
| Apollo 2.0 | 2018.01 | 城市道路简单场景，激光雷达感知，高精地图集成 |
| Apollo 3.0 | 2018.07 | 低速园区全自动驾驶，Valet Parking 代客泊车 |
| Apollo 5.0 | 2019.07 | 城市开放道路 L4 级，ANP（Apollo Navigation Pilot）高速辅助 |
| Apollo 6.0 | 2020.09 | 引入云端训练闭环，数据驱动模型迭代 |
| Apollo RT6 | 2022.07 | 第六代无人驾驶汽车，成本降至 25 万元，全无人商业运营 |


## 系统架构

Apollo 采用"车端软件栈 + 云端服务"的双层架构：

```
┌──────────────────────────────────────────────────────────┐
│                      云端服务                             │
│   高精地图更新 │ 仿真测试（Apollo Sim）│ 数据标注 │ OTA  │
└─────────────────────────┬────────────────────────────────┘
                          │ 5G / V2X
┌─────────────────────────▼────────────────────────────────┐
│                    车端软件栈                              │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐             │
│  │  感知     │──►│  预测    │──►│  规划    │             │
│  │Perception│   │Prediction│   │ Planning │             │
│  └──────────┘   └──────────┘   └──────────┘             │
│       ▲                               │                  │
│  ┌────┴─────┐   ┌──────────┐   ┌─────▼────┐            │
│  │   定位   │   │ 高精地图  │   │  控制    │             │
│  │Localiz.  │   │  HD Map  │   │ Control  │             │
│  └──────────┘   └──────────┘   └──────────┘             │
│                                                          │
│               Cyber RT 中间件                            │
└──────────────────────────────────────────────────────────┘
```

**Cyber RT 中间件**（Apollo 3.5 后替代 ROS）的核心特性：
- **去中心化通信**：组件间直接通信，消除中央调度瓶颈
- **协程调度**：基于协程（Coroutine）的任务调度，减少线程切换开销，降低延迟
- **组件（Component）化**：模块独立编译、独立启动、独立替换，便于开放生态接入

## 感知模块

Apollo 感知模块整合多传感器数据，输出统一目标列表（Object List）：

- **LiDAR 3D 检测**：基于 PointPillars、CenterPoint 等高效网络，检测车辆、行人、骑行者的 3D 边界框
- **BEV 视觉感知**：多摄像头特征投影至鸟瞰视角，实现统一的 2D/3D 目标检测
- **传感器融合**：时间同步（硬件时钟同步，精度 < 1 ms）+ 外参标定后进行特征级融合，输出融合目标列表及不确定性估计


## 高精地图与定位

Apollo 采用双层高精地图架构：

| 地图层 | 内容 | 用途 |
| --- | --- | --- |
| 点云地图 | 高密度 3D 点云 | LiDAR 实时扫描匹配定位 |
| 矢量 HD Map | 车道几何、信号灯位置、限速、坡度 | 路径规划与决策 |

**定位方案：** GNSS/RTK + IMU + LiDAR 点云匹配的多传感器融合，实现 **5–10 cm** 级横向定位精度。


## Robotaxi 商业化（萝卜快跑）

百度旗下 Robotaxi 品牌**萝卜快跑（Apollo Go）** 是全球规模最大的 L4 级自动驾驶出行服务：

**里程碑事件：**
- **2020 年**：北京首批发放 L4 自动驾驶路测牌照
- **2022 年**：重庆、武汉获批无安全员测试
- **2023 年 8 月**：武汉开放**全无人驾驶**商业收费运营（全球首例城市规模）
- **2024 年**：
  - 运营城市扩展至 10+ 个（北京、上海、广州、深圳、武汉、重庆等）
  - 累计服务订单超 **600 万次**
  - 武汉日均订单峰值超 **2 万单**，引发社会广泛讨论

**成本下降路径：**

| 车型代 | 上市年份 | 单车成本 | 备注 |
| --- | --- | --- | --- |
| 第三代（L4） | 2019 | > 100 万元 | 大量进口 LiDAR |
| 第五代 | 2021 | 约 48 万元 | 国产 LiDAR 比例提升 |
| RT6（第六代） | 2022 | 约 25 万元 | 双 Orin，全自研传感器 |

Apollo 开放平台已构建起涵盖 **210+ 家合作伙伴**的生态系统：

| 合作类型 | 代表机构 |
| --- | --- |
